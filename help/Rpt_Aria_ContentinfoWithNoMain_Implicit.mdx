## Who does this affect?

* Blind people using screen readers

## Why is this important?

A element with `"contentinfo"` role is an area that identifies common information at the bottom of each page, typically called the footer of the page. It includes information such as copyrights and links to privacy and accessibility statements. A footer without any main content within a `"main"` role is not permited. The HTML `<footer>` element has an implicit role of `"contentinfo"` when it is a direct descendent of the `<body>` element.
</Column>

## A element with `"contentinfo"` role is present without an element with a `"main"` role
A element with `"contentinfo"` role is only permitted with an element with `"main"` role

[IBM 2.4.1 Bypass Blocks](http://www.ibm.com/able/guidelines/ci162/bypass_blocks.html) | [WAI-ARIA 1.1 Contentinfo Role](https://www.w3.org/TR/wai-aria-1.1/#contentinfo)

<div id="locSnippet"></div>

## What to do

* Add a `<main>` element that has an implicit role of `"main"` to enclose the main content of the Web page.


For example:

<CodeSnippet type="multi" light={true}>&lt;body&gt;
    &lt;div role="main"&gt;
      ...
    &lt;/div&gt;
    &lt;div role="contentinfo"&gt;
	    ...
    &lt;/div&gt;
&lt;/body&gt;</CodeSnippet>
