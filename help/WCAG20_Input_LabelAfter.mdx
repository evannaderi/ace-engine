## Who does this affect?

 * Blind people using screen readers
 * People with low vision using screen magnification

## Why is this important?

Applications generally place the labels for radio buttons and check boxes after the input control. This practice causes users to expect label placement to be consistent. When label placement does not follow this convention, users can become disoriented.

</Column>

## Label text is before its associated checkbox or radio button
Check boxes and radio buttons must have a label after the input control

[IBM 1.3.1 Info and Relationships](https://www.ibm.com/able/guidelines/ci162/info_and_relationships.html) | [IBM 4.1.2 Name, Role, Value](https://www.ibm.com/able/guidelines/ci162/name_role_value.html) | [WCAG 2.1 technique H44](https://www.w3.org/WAI/WCAG21/Techniques/html/H44)

<div id="elementLocation">
</div>

## What to do

* Place the text of the label immediately after the checkbox or radio button. For example:
```
<input id="chk1" name="chkgroup" type="checkbox"><label for="chk1">Option 1</label>
<label><input name="chkgroup" type="checkbox">Option 1</label>
```
